ПРОБЛЕМЫ:

- метод startCommand вынести в отдельный класс executor (который в 6-ой лабе окажется на серверной части).

- сам метод будет принимать аргумент - класс Request, который хранит всю информацию о названии команды, ее типе и аргументах. все аргументы строго своего типа (long, HumanBeing, string и тд) не object!

- метод возвращает результат своей работы обернутый в класс Response - который хранит в себе результат выполнения команды: строка, статус выполнения + list<humanBeing>*

- для классов request сделать иерархию - родительский абстрактный класс (с полем commandName), а дальше реализации с каждой своим типом команды - без аргументов, с аргументом id, с аргументом human being и тд. а каждый класс команды принимает только свой тип запроса и работает с ним
